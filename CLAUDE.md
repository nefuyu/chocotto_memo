# Chocotto Memo App

## プロジェクト概要
- シンプルなオフライン専用メモアプリ。
- 技術スタック: Flutter, SQLite (sqflite)

## 開発ガイドライン
1. **TDD (Test Driven Development)**: 
   - 実装コードを書く前に、必ず `test/` フォルダ内に仕様に基づいたテストコードを作成せよ。
   - `flutter test` を実行し、テストが通ることを確認しながら実装を進めること。
2. **ブランチ運用**: 
   - 作業開始時に必ず `task/issue-[ID]` の形式で新ブランチを作成せよ。
   - 完了後はそのブランチにコミットせよ。
3. **プライバシー**: 
   - オフライン専用。外部API通信やFirebase等は一切使用せず、ローカル保存のみで完結させること。
4. **コミュニケーションと中断ルール**:
   - Issueの内容に矛盾がある、または仕様が不足している場合は、実装を強行しないこと。
   - 疑問点は `gh issue comment [ID] -b "質問内容"` を使ってIssueに投稿せよ。
   - 質問を投稿した後は、作業を一旦停止（待機）し、ユーザーの回答を待つこと。

## よく使うコマンド
- テスト実行: `flutter test`
- アプリ起動: `flutter run`
- 依存関係更新: `flutter pub get`
