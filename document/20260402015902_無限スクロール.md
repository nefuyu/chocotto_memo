# 全メモ一覧の無限スクロール実装（100件ずつ分割読み込み）

## Issue概要

Issue #26：全メモ一覧画面に無限スクロール（ページネーション）を実装する。
スクロールが末尾に近づいたら次の100件を追加読み込みし、ローディングインジケータを表示する。

## 実装方針

- `DatabaseService.getAll()` に `limit` / `offset` オプションパラメータを追加。
- `HomeScreen` に `ScrollController` を導入し、末尾200px手前で次ページを読み込む。
- 1ページあたり100件（`_pageSize = 100`）。
- 読み込み中は `_isLoadingMore` フラグを立て、リストの末尾に `CircularProgressIndicator` を表示。
- `_hasMore` フラグで全件読み込み済みかを判定し、不要な追加読み込みを防ぐ。

## クラス/関数構成

### `lib/services/database_service.dart`

| 変更点 | 説明 |
|--------|------|
| `getAll({int? limit, int? offset})` | `limit`/`offset` を追加。既存の引数なし呼び出しは後方互換を維持。|

### `lib/screens/home_screen.dart`

| 追加要素 | 説明 |
|----------|------|
| `_pageSize` | 1ページあたりの件数（100） |
| `_isLoadingMore` | 追加読み込み中フラグ |
| `_hasMore` | 次のページが存在するかフラグ |
| `_scrollController` | スクロール末尾検知用 |
| `_onScroll()` | 末尾200px以内で `_loadMore()` を呼び出す |
| `_loadMore()` | 次ページを取得して `_memos` に追記 |

`_loadMemos()` は `limit: _pageSize, offset: 0` で初期ロード。
`_loadMore()` は `offset: _memos.length` で次ページを取得。

## テスト方針

### `test/database_service_test.dart`（グループ: DatabaseService ページネーション）

1. `limit` 指定で指定件数だけ返る
2. `offset` 指定で先頭をスキップして返る
3. `offset` が総件数以上のとき空リストを返す
4. `limit` なしの場合は全件返る

### `test/home_screen_test.dart`（グループ: HomeScreen - 無限スクロール）

1. メモが100件以下の場合はローディングインジケータが表示されない
2. スクロール末尾到達時に次のページが追加読み込みされる
3. 全件読み込み済みの場合は追加読み込みされない

### `test/fake_database_service.dart`

- `getAll()` に `limit`/`offset` 対応を追加。
- `getAllCallCount` カウンタを追加（テストから呼び出し回数を検証するため）。

## 既知の制約

- `_loadMemos()`（全リセット）と `_loadMore()` を並行して呼び出すと件数がずれる可能性があるが、UIでは同時発生しない構造になっている。
- 削除・編集後に `_loadMemos()` でリセットするため、削除後は先頭から再取得される。
